JavaScript essencial: tipos, escopo e o DOM

Diogo Gomes  ·  20 de Junho, 2026  ·  3 min de leitura

JavaScript é a linguagem que dá vida à página, mas tem armadilhas que pegam todo iniciante: por que 0 == '' é verdadeiro? Por que uma variável "vaza" para fora do bloco? Dominar tipos, escopo e o DOM é o que transforma código que funciona por acaso em código que você entende.

Tipos: primitivos vs referência

Valores primitivos (number, string, boolean, null, undefined, symbol, bigint) são copiados por valor. Já objetos, arrays e funções são manipulados por referência: a variável guarda um endereço, não o conteúdo. Pense num primitivo como um bilhete escrito; copiar é reescrever. Um objeto é um endereço de casa; copiar a variável é só anotar o mesmo endereço — as duas apontam para a mesma casa.

let a = 1; let b = a; b = 2;
// a continua 1 (cópia de valor)

let o = { n: 1 }; let p = o; p.n = 2;
// o.n agora é 2 (mesma referência)

== vs === e coerção

O == faz coerção: tenta converter os tipos antes de comparar, gerando surpresas como 0 == '' ou null == undefined serem true. O === compara valor e tipo, sem conversão. A regra de ouro: use sempre ===.

0 == '';        // true  (coerção!)
0 === '';       // false (tipos diferentes)
'5' == 5;       // true
'5' === 5;      // false

var, let, const: escopo e hoisting

let e const têm escopo de bloco (vivem entre { }); var tem escopo de função e "vaza". O hoisting é o JS "içar" declarações para o topo do escopo: var é içada como undefined, enquanto let/const ficam numa "zona morta temporal" e dão erro se acessadas antes da declaração. Prefira const, use let quando precisar reatribuir, e evite var.

Closures: a função que lembra

Uma closure acontece quando uma função "lembra" das variáveis do escopo onde foi criada, mesmo depois que esse escopo terminou. É como uma mochila que a função carrega com o ambiente dela dentro.

function contador() {
  let n = 0;
  return () => ++n;
}
const c = contador();
c(); // 1
c(); // 2  (lembra de n)

O DOM: a página como árvore

O DOM (Document Object Model) é a representação da sua página HTML como uma árvore de objetos que o JavaScript pode ler e alterar. Cada tag vira um nó; você seleciona nós e muda texto, classes, atributos. Não é o HTML em si — é a versão viva dele na memória do navegador.

const btn = document.querySelector('#salvar');
btn.textContent = 'Salvar agora';

Eventos: reagindo ao usuário

O usuário clica, digita, rola — e o JS reage via eventos. Você registra um "ouvinte" e ele dispara quando o evento acontece.

btn.addEventListener('click', () => {
  console.log('cliquei!');
});
Tipos, escopo e closures não são curiosidades acadêmicas: são exatamente o que explica os bugs mais comuns de quem está começando.

É comum em entrevista pedir para você prever a saída de uma comparação com ==, explicar o que é uma closure ou diferenciar valor de referência. No dia a dia, isso aparece quando um array "muda sozinho" (era a mesma referência) ou um setTimeout dentro de um loop imprime o número errado.

Conclusão

Entender que primitivos copiam e objetos referenciam, sempre comparar com ===, respeitar o escopo de bloco e enxergar o DOM como árvore — esse é o alicerce do JS. Com closures no bolso, você já está pronto para o passo que confunde até gente experiente: o event loop.
