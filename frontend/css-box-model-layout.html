CSS sem medo: box model, flexbox e grid

Diogo Gomes  ·  22 de Junho, 2026  ·  3 min de leitura

CSS assusta no começo porque parece imprevisível: você muda uma propriedade e a página inteira se mexe. A boa notícia é que quase toda essa "mágica" se resume a três ideias — o box model, o flexbox e o grid. Entendendo elas, o CSS deixa de ser tentativa e erro e vira algo que você controla.

Tudo é uma caixa: o box model

Todo elemento na tela é uma caixa retangular composta por quatro camadas, de dentro para fora: content (o conteúdo), padding (espaço interno), border (borda) e margin (espaço externo). Pense numa moldura de quadro: a foto é o content, o paspartur é o padding, a moldura é a border e o espaço até o próximo quadro na parede é a margin.

.card {
  width: 200px;
  padding: 20px;
  border: 2px solid;
}
/* Largura real = 200 + 20*2 + 2*2 = 244px (!) */

box-sizing: border-box salva sua sanidade

Por padrão, width conta só o content — padding e border somam por cima, e a caixa estoura. Com box-sizing: border-box, a largura passa a incluir padding e border. É tão essencial que quase todo projeto aplica isso globalmente.

* { box-sizing: border-box; }
.card { width: 200px; padding: 20px; }
/* Agora a largura real É 200px */

display: como a caixa se comporta

O display define o fluxo: block ocupa a linha inteira (como um parágrafo), inline fica na mesma linha (como um <span>) e ignora width/height, e flex/grid transformam o elemento num container que organiza seus filhos.

Flexbox: layout em uma dimensão

Flexbox organiza itens ao longo de um eixo por vez. Você define o eixo principal com flex-direction (linha ou coluna) e o eixo cruzado é o perpendicular. justify-content alinha no eixo principal; align-items alinha no cruzado. Perfeito para barras de navegação, listas de botões, qualquer fileira.

.barra {
  display: flex;
  justify-content: space-between;
  align-items: center;
}

Grid: layout em duas dimensões

Quando você precisa de linhas e colunas ao mesmo tempo — uma galeria, um dashboard, o layout da página inteira — Grid resolve rápido. Você desenha a grade e encaixa os itens.

.galeria {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
  gap: 16px;
}

Flex ou Grid? E como centralizar

A regra prática: uma direção, use flex; duas direções, use grid. Não é dogma — muita gente faz layout inteiro com flex —, mas é o ponto de partida certo.

CritérioFlexboxGrid
Dimensões1D (linha OU coluna)2D (linhas E colunas)
Melhor paraComponentes, barrasLayout de página, grades
Define forma a partir deConteúdoContainer

E o eterno "como centralizo um elemento?". Com flex, três linhas:

.centro {
  display: flex;
  justify-content: center; /* horizontal */
  align-items: center;     /* vertical */
}
Centralizar virou trivial: o difícil não é mais "como", é lembrar que justify-content e align-items mudam de eixo conforme o flex-direction.

É comum em entrevista pedir exatamente para explicar o box model, a diferença entre content-box e border-box, ou como centralizar vertical e horizontalmente. No dia a dia, é o que você usa em todo componente que monta.

Conclusão

O box model explica o tamanho das coisas, o flexbox organiza em uma dimensão e o grid em duas. Com border-box ligado e essas duas ferramentas no cinto, o CSS deixa de ser sorte. O próximo passo é dar vida à página com JavaScript.
